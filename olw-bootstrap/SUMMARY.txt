OVERVIEW
--------
Automatically generates comprehensive Operational Language Wiki (OLW) systems
from domain knowledge and institutional context. Creates interconnected wiki
pages following WikiPage v1 specification.

QUICK START
-----------
Use the olw-bootstrap skill to create a wiki from [source] outputting to [dir]

Example:
  Use olw-bootstrap to generate a wiki from globals/metadata/promptql-config.hml

WHAT IT CREATES
---------------
✓ 15-25 interconnected wiki pages per domain
✓ WikiPage v1 YAML format
✓ Cross-linked with wiki:// protocol
✓ Semantic aliases for AI search
✓ Structured definitions, details, and sections
✓ Institutional terminology consistency

COVERAGE AREAS
--------------
1. Core Entities (1-2 pages)      - Main organization/system
2. Classifications (2-5 pages)    - Segments, tiers, types
3. Products/Services (2-4 pages)  - Offerings, solutions
4. Metrics (3-6 pages)            - KPIs, calculations, rates
5. Processes (2-4 pages)          - Operations, workflows
6. Events/Patterns (1-3 pages)    - Cycles, triggers
7. Strategic (2-4 pages)          - Initiatives, rules
8. Data Concepts (2-4 pages)      - Fields, statuses

INPUT OPTIONS
-------------
• Config files (YAML, HML, JSON)
• Documentation (Markdown, text)
• Direct description
• Hybrid (file + additional context)

OUTPUT FORMAT
-------------
kind: WikiPage
version: v1
definition:
  title: "Concept Name"
  definition: |
    Brief definition
  details: |
    Expanded info with wiki://cross-links
  aliases:
    - "Alternative Name"
  sections:
    - title: "Section Name"
      content: |
        Detailed content

QUALITY STANDARDS
-----------------

PERFORMANCE
-----------
